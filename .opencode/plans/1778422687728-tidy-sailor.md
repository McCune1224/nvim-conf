# Lualine Redesign Plan

## Goal
Create 5 standalone lualine style files the user can swap between, plus backup the current config. All styles share the same custom helper functions (mode icon+text, git, diagnostics, position, etc.) but have distinct separator/spacing/padding configurations.

## Current Setup
**File:** `lua/plugins/lualine.lua` (246 lines)

Has: mode icon, git info, file path+status, diagnostics, position, progress, tab number, aerial breadcrumbs in tabline, 6 extensions (`theme = 'auto'`, `globalstatus = true`, empty separators).

## Execution Plan

### Step 1: Backup Current
- Copy `lua/plugins/lualine.lua` → `lua/plugins/lualine.lua.bak2`
- The existing `.bak` is an older simpler version (keep it too)

### Step 2: Create Shared Helpers Module (optional improvement)
Option A: Extract shared functions into `lua/config/lualine_helpers.lua` so each style just requires it (DRY).
Option B: Keep each file self-contained (duplicate the 80 lines of helpers in each).
→ **Use Option B** — each file is fully self-contained so the user can tinker/rename individually without dependencies.

### Step 3: Create 5 Style Files

All 5 files include these shared components:
- `mode_icons` table (unchanged)
- `get_mode_icon()` + new `get_mode_text()` (returns "NORMAL", "INSERT", etc.)
- `filepath()`, `file_status()`, `git_info()`, `diagnostics()`, `tab_number()`, `position()`, `progress()`
- Aerial breadcrumbs in tabline (same as current)
- Same extensions list
- `theme = 'auto'`, `globalstatus = true`

Each file has a UNIQUE `lualine.setup()` configuration:

### Step 4: Update init.lua
The current `init.lua` has:
```lua
require 'plugins.lualine'
```

Keep this as-is. The user will manually copy their chosen style to `lualine.lua` to activate it. Or use a simple file rename.

Better approach: Update `init.lua` to load from a configurable name. The user can set which style is active:
```lua
-- Change this to switch lualine styles:
-- 'lualine', 'lualine-style-a-block-chunks', 'lualine-style-b-clean-rail', etc.
vim.g.lualine_style = 'lualine-style-c-floating-glass'
require('plugins.' .. vim.g.lualine_style)
```

This is the cleanest — no file renaming, just change one variable.

**Update Step:** Modify `init.lua` line 32 from:
```lua
require 'plugins.lualine'
```
to:
```lua
-- Active lualine style (change to switch between styles):
-- Options: lualine, lualine-style-a-block-chunks, lualine-style-b-clean-rail,
--          lualine-style-c-floating-glass, lualine-style-d-terminal-pills,
--          lualine-style-e-bento-grid
vim.g.lualine_style = 'lualine-style-c-floating-glass'
require('plugins.' .. vim.g.lualine_style)
```
